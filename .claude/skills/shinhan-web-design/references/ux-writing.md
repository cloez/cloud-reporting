# Shinhan — UX Writing 가이드라인

신한 쏠(SOL) UX Writing 가이드라인을 기반으로, Shinhan UI의 모든 텍스트 작성 시 적용한다.

---

## 목차

1. 보이스앤톤
2. 라이팅 기본원칙 (명확하게 / 간결하게 / 일관되게 / 고객관점으로)
3. 콘텐츠 유형별 글쓰기 (버튼 / 바텀시트 / 레이블 / 툴팁 / 텍스트필드 / 팝업 / 토스트 / 메시지)
4. 서술어 톤앤매너
5. 표기 규칙 (문장부호 / 날짜·시간 / 금액 / 영업일 등)
6. 권장용어 및 금지표현

---

## 1. 보이스앤톤

### Voice (브랜드 고유 어조 — 일관 유지)

| 보이스 | 설명 |
|------|------|
| **Cheerful** (경쾌하고 즐거운) | 익숙하고 일상적인 어휘, 긍정적·친근한 표현, 진지하고 사무적인 표현 대신 즐거움을 더하는 표현 |
| **Straightforward** (꾸밈없이 명료한) | 고객이 주체가 되는 능동 표현, 쉬운 용어와 익숙한 표현, 과장·추상 없이 명료하고 구체적으로 |
| **Beneficial** (고객 혜택 강조) | 유익한 금융 상품·서비스 정보 제공, 혜택을 부담 없이 전달, 불편 상황에 분명한 해결책 제시 |

### Tone (상황별 유동 말투)

| 톤 | 설명 | 이렇게 쓰세요 | 이렇게 쓰지 마세요 |
|---|------|----------|-------------|
| **Friendly** (친근한) | 익숙하고 친근한 언어, 자연스러운 대화체 | 대화체 표현, 일상어, 부드러운 권유·제안 어투 | 전문용어, 과도한 높임말, 딱딱한 문어체 |
| **Clear** (쉽고 간결한) | 한 번에 빠르게 파악, 의사결정 지원 | 한 문장에 한 메시지, 핵심만 짧고 명확하게 | 지시어, 추상적 표현, 용어 반복, 줄임말 |
| **Informative** (유익한) | 실질적·구체적 정보 제공 | 숫자·데이터 활용, 무엇을 어떻게 해야 하는지 안내 | 과도한 주의사항 반복, 불필요 정보, 기준 없는 용어 혼용 |
| **Caring** (배려하는) | 고객 관점, 공감과 배려 언어 | 능동적·긍정적 표현, 눈높이 맞는 설명, 해결책 명확 제시 | 지시적·사무적·권위적 표현, 강한 부정, '실패' 표현 |

### 텍스트 기능별 톤 매핑

| 텍스트 기능 | 적용 보이스 | 적용 톤 | 적용 영역 |
|---------|---------|-------|--------|
| 정서 교감 | Cheerful | Friendly | 첫인사, 개인화 메시지, 마케팅 카피, 서비스 이용 유도 |
| 정보 전달 | Straightforward | Clear + Informative | 상품·서비스 소개, 이용방법, 단순 안내 |
| 설득 | Beneficial | Caring | 회원가입/탈퇴/동의, 상품·서비스 가입 프로세스 |
| 책무 | 보이스 적용 안 함 | 사실 기반 정중한 톤 | 서비스 약관, 제한사항, 유의사항 |

## 2. 라이팅 기본원칙

### 원칙 1: 명확하게

- 금융용어·내부용어는 누구나 이해할 수 있도록 쉽게 표현
- 어려운 한자어는 우리말 또는 더 쉬운 한자어로 대체
- 한자·영어를 우리말 없이 단독 표기하지 않음
- 줄임말·지시어(여기, 이곳, 당사, 본, 상기, 하단 등) 사용 금지

**한자어 순화 주요 목록:**

| 한자어 | 순화어 | 한자어 | 순화어 |
|------|------|------|------|
| 성명 | 이름 | 당월/전월 | 이번달/지난달 |
| 자택 | 집 | 익일/익월 | 다음날/다음달 |
| 잔여 | 남은 | 존재하다 | 있다 |
| 향후 | 앞으로 | 일괄 | 한 번에, 한꺼번에 |
| 상이하다 | 다르다 | 고지/통보 | 안내/알림 |

**한자 접두사 풀어쓰기:**

| 접두사 | 사용 금지 | 권장 |
|-------|---------|-----|
| 기(旣)- | 기납부, 기가입, 기등록 | 이미 납부한, 이미 가입한, 기존 등록 |
| 미(未)- | 미등록, 미신청 | 등록하지 않은, 신청하지 않은 |
| 비(非)/불(不)- | 비회원, 불가 | 회원만, ~할 수 없습니다 |

### 원칙 2: 간결하게

- 긴 문장은 짧게 나누기 (한 문장 30~40글자, 7~10어절 이내)
- 중복·불필요 표현 제거 ('진행', '처리', '성공적으로', '정상적으로', '완료' 등 군더더기 지양)
- 번역투·피동형 간결하게 다듬기 ('~을 통해', '~에 대하여', '~에 의해', '되어지다' 등 지양)
- 지나친 높임말 사용 금지 ('~시~'는 마지막 서술어에 한 번만, '~오니' 대신 '~하므로/~하니')
- 사과·양해 표현은 큰 피해/불편 시에만 한정 사용

### 원칙 3: 일관되게

- 같은 의미 용어는 하나로 통일
- 같은 화면·프로세스 단계에서 용어·표현 일관 유지
- 문장 톤앤매너 통일 (서술형/명사형 혼용 금지)
- 기호·문장부호·날짜·시간·금액 등 표기 통일

**용어 통일 주요 목록:**

| 사용 금지 | 통일 용어 |
|---------|---------|
| 성명, 성함 | 이름 |
| 핸드폰, 휴대전화, 폰 | 휴대폰 |
| 암호, 패스워드 | 비밀번호 |
| PUSH, 앱푸시 | 푸시알림 |
| 공인인증서 | 공동인증서 |

### 원칙 4: 고객 관점으로

- 기업 관점 표현 → 고객 관점 표현 ('통보'→'안내', '거래'→문맥에 맞게, '제공'→'이용/조회')
- 고객 주체 능동 표현, 혜택 담은 표현 사용
- '불가능', '불가' 등 강한 부정 금지 → '~할 수 없습니다'
- 이중 부정 금지
- '실패' 표현 금지 → '~하지 못했습니다' + 대처 방법 안내

## 3. 콘텐츠 유형별 글쓰기

### 버튼

**Single 버튼:**
- 명사형으로 간결하게, 되도록 4글자 이내
- [확인], [취소], [완료], [로그인], [다음에]에는 '~하기' 붙이지 않음
- 고객 행동 유도 시 12글자·4어절 이내까지 허용

**Multiple 버튼:**
- 최대한 간결한 명사형, '~하기' 형태를 양쪽에 나란히 쓰지 않음
- 오른쪽 버튼은 사용자 행동을 표현한 '액션'으로 (7글자 이내, 2어절)
- 부정 버튼: [취소], [닫기], [다음에]

**예/아니요 버튼:**
- 취소 의사, 해지/중단 등 불편 상황, 단순 yes/no 질문에만 사용
- '아니오' (X) → '아니요' (O)

**여부 버튼:**
- 프로세스 의사결정: 부정 → 긍정 순 배치, '액션안함' / '액션' 형태
- 동일 레벨 단순 선택: 긍정 → 부정 순 배치
- 한자 접두사 금지: 비신청(X) → 신청안함(O)

**CTA 버튼:**
- '~하기' 기본형, 고객 행동 또는 가치(혜택)에 집중
- 대화형: '해요체', 대화 흐름에 맞는 표현

**링크 버튼:**
- '~하기' 기본형, 최대 12글자 이내 (7자 이상이면 '~하기' 생략)
- '더보기', '자세히보기' 모두 붙여 쓰기
- 질문형은 1인칭 시점

### 바텀시트

**타이틀:** 핵심만, 서비스 안내는 '~안내', 요청은 서술형(마침표 생략), 옵션은 명사형
**본문:** 서비스 안내는 '해요체'/명사형, 요청은 '해요체', 옵션은 명사형. 유의사항은 '하십시오체'

### 레이블

**전체 메뉴:**
- 상위 메뉴: 5글자 이내, 띄어쓰기 없이
- 하위 메뉴: 12글자 이내, 4글자 초과 시 의미 단위로 한 번만 띄어 쓰기

**상품/서비스 메뉴:** 5글자 이내, 2개 이상은 빗금(/) 사용

### 툴팁

- 최대한 짧게, 도움 되는 정보만
- 기본형: '해요체', 나열형: 명사형
- 하이퍼링크 넣지 않음

### 텍스트필드

**레이블+입력창:** 중복 금지, 액션형 명사 사용
**하단 도움말:** 1줄, '하십시오체', 행동 요청은 '~해주세요', 긍정형 지향

### 팝업

- '하십시오체' 기본, 상황 설명 → 의사 확인/행동 요청 순서
- 질문: '~하시겠어요?', 요청: '~해주세요'
- 기본형: 타이틀 없이 3줄 이내
- 타이틀형: 목적 중심 짧고 간결하게, 서술형 타이틀에는 마침표 생략

### 토스트

- 컨펌형(능동): 17자 이내 '하십시오체', '~를 ~했습니다' (능동태)
- 컨펌형(피동): 시스템 주체일 때만 피동형 허용
- 요청형: 31자 이내 '하십시오체', '~해주세요'
- 말풍선: 17자 이내 '해요체'
- '처리', '완료', '성공적으로', '정상적으로', '실패' 사용 금지

### 헤드 메시지

- 쉬운 일상어 + '해요체', 2줄 31글자 이내 (보조 설명 2줄 추가 가능)
- 친근하고 경쾌한 표현
- 의문문: '~할까요?', '~하실래요?' / 부정적 질문: '~하시겠어요?'

### 완료 메시지

- 상단: '명사형 종결' — '액션형 명사 + 완료' (띄어쓰기 함, 느낌표 금지)
- 하단: '해요체' 2줄 이내, 고객 관점 표현, 상단과 중복 금지

### 거절 메시지

- '하십시오체', 사유 2줄 이내로 안내
- '죄송합니다' 등 사과 표현 지양 (심각한 피해 시에만 한정)

### 오류 메시지

- '하십시오체', 현재 상태 → 발생 원인 → 해결 방법 순
- 오류코드·개발 언어 노출 금지
- '거래', '처리', '유효하다', '당행', '실패' 등 금지

### 이탈방지 메시지

- '하십시오체', 불이익부터 나열하지 않음
- 혜택·장점 강조 (이때 '해요체' 허용)
- 부정 버튼에 [취소] 대신 [계속 이용] 등 능동적 버튼

### 유의사항

- '하십시오체', 사실 정보만
- 타이틀: '알아두세요' 통일 (대출 등 중요 고지는 '유의사항')
- 주요 항목: 불릿(·), 고지성 문구: 참고표(※) + 서술형 종결
- 부속 항목: 숫자 또는 붙임표(-), 명사형 종결
- 요청: '~해주세요'

## 4. 서술어 톤앤매너

| 상황 | 서술어미 | 예시 |
|----|--------|-----|
| 정보 안내 (이용안내, 유의사항) | '~ㅂ니다' (하십시오체) | 이미 등록된 쿠폰입니다. |
| 흥미 유발 (서비스/상품 소개, 설득) | '~요', '~해보세요', '~할까요?' (해요체) | 든든한 노후 준비, 개인형 퇴직연금으로 준비해보세요. |
| 행동 요청 (입력/선택/검색 등) | '~해주세요', '~해보세요' | 올바른 비밀번호를 입력해주세요. |
| 고객 의사 확인 (팝업 등) | '~하시겠어요?' | 이체를 중단하시겠어요? |
| 가입/참여 유도 (마케팅, 이벤트) | 명사형 종결 '~하기' | 친구에게 적금 추천하기 |

### 높임법

**해요체 적용:** 마케팅 카피, 헤드메시지, 말풍선, 문장형 타이틀/메뉴명, 상단 문구, 설문, 가벼운 요청/권유
**하십시오체 적용:** 팝업/토스트 문구, 유의사항, 약관/고지 문구

- '~하시기 바랍니다' → '~해주세요'
- '-시-'는 마지막 서술어에 한 번만
- '~하셨습니까?' → '~하시겠어요?'

## 5. 표기 규칙

### 문장부호

| 부호 | 용도 | 예시 |
|----|-----|-----|
| 마침표(.) | 문장 끝맺음 | 영업점에 방문해 변경해주세요. |
| 쉼표(,) | 짧게 끊어 읽기, 자릿점 | 2,000,000원 |
| 빗금(/) | 대응·대립 나열 | 홈페이지/모바일앱 |
| 소괄호(()) | 앞 단어에 붙여 씀, 뒤 명사는 띄어 쓰고 조사는 붙여 씀 | 마감시간(LT시간) 이내에만 |
| 대괄호([]) | 메뉴명·버튼명·경로 표기 | [타행계좌 출금결과조회] |
| 닫는 홑화살괄호(>) | 경로 표시 (앞뒤 띄어 씀) | [금융서비스 > 법원업무 > 계좌환급 조회] |
| 물결표(~) | 기간·범위 (앞뒤 붙여 씀), 단위는 뒤에 한 번만 | 2.5~3.6% |
| 별표(*) | 별도 참고정보 | * 신한디에스에 등록된 휴대폰번호가 없습니다. |
| 참고표(※) | '알아두세요' 내 중요정보 | ※ 중도해지 시 이자율이 달라질 수 있으니 주의해주세요. |

### 날짜

- 기본: YYYY.MM.DD (예: 2022.01.25)
- 공간제약: YY.MM.DD (예: 22.01.25)
- 서술형: 년/월/일 한글 표기, 한 자리 숫자 앞에 0 제거 (예: 2022년 5월 6일)
- 요일: 소괄호 안 앞 한 글자 (예: 2022.01.24(월))

### 시간

- 24시간 표기법: HH:MM~HH:MM (예: 09:00~18:00)
- '시', '부터', '까지' 함께 쓰지 않음
- 단, 분/초 없이 시간만 안내 시 12시간 표기 허용

### 기간

- 같은 연도: YY.MM.DD~MM.DD (예: 22.04.15~05.14)
- 다른 연도: YY.MM.DD~YY.MM.DD

### 금액

- 세 자리마다 쉼표, 숫자 다음에 '원' (예: 10,000원)
- 천원, 만원 등 한글 단위 붙여 씀 (예: 500만원, 2,500만원)
- 만 단위부터 한글 표기
- 외환: 통화코드+금액(한글 통화단위) (예: USD 500(미국 달러))

### 이자율·퍼센트

- 소수점 이하 1~2자리 (예: 연 2.53%)
- '이율' 사용 금지 → '이자율' 또는 합성어에서 '금리'
- 범위: 뒤에만 % 한 번 (예: 1.6~3.8%)

### 영업일

- 통일 용어: '영업일'
- 전날(영업일 기준), 다음날(영업일 기준)
- 이용시간: 평일 09:00~18:00(토/일/공휴일 제외)
- 영업일 아닌 날: 토/일/공휴일

### 전화번호

- 담당처 이름 뒤 소괄호 (예: 고객센터(1599-8000))
- 특수문자(☎, ☏) 사용 금지

### 버전명·OS

- iOS, 안드로이드, 윈도우 (한글 우선)
- 버전 뒤에 '버전' 한글 별도 표기 금지 (예: 안드로이드 9.0 이상)

## 6. 권장용어 및 금지표현

### 영어/외래어 우리말 표현

| 사용 금지 | 권장 | 사용 금지 | 권장 |
|---------|-----|---------|-----|
| App | 앱 | 에러 | 오류 |
| SMS | 문자메시지 | 프로세스 | 절차 |
| FAX | 팩스 | 리스트 | 목록 |
| E-Mail | 이메일 | 패스워드 | 비밀번호 |
| SP | 스마트폰 | 런칭 | 출시 |
| or | 또는 | 어플리케이션 | 앱, 애플리케이션 |
| ex. | 예, 예시 | 메세지 | 메시지 |

- OTP, ARS, ATM은 단독 사용 허용
- 영어 약어 전문용어는 우리말 우선 + 괄호 안에 영어 (예: 총부채원리금상환비율(DSR))

### 지시어 사용 금지

| 사용 금지 | 권장 |
|---------|-----|
| 본 서비스, 본 상품 | 이 서비스, 이 상품 (또는 구체적 이름) |
| 당행 | 신한디에스 |
| 타행 | 다른은행 |
| 상기, 상단 | 위 |
| 하기, 하단 | 아래 |

### 차별 표현 금지

- 성별·연령 차별: 효자상품→인기상품, 유모차→유아차
- 직업 비하: 청소부→환경미화원, 수위→경비원
- 장애인 차별: 장애자→장애인, 정상인→비장애인, 맹인→시각장애인

### 신한 쏠(SOL) 띄어쓰기 규칙 (내부 약속)

**붙여 쓰기:**
- 본용언+보조용언: 입력해주세요, 신청해주세요
- ~보기/~가기: 더보기, 바로가기, 자세히보기
- ~날/~주/~달: 전날, 다음날, 이번주, 다음달
- OO안함(버튼명): 신청안함, 동의안함
- ~방법/~정보/~내역/~금액: 이용방법, 계좌정보, 이체내역, 이용금액
- ~원: 100만원, 1천만원

**띄어 쓰기:**
- 이름 + 님: 김신한 님
- 총 + 명사: 총 한도, 총 금액
