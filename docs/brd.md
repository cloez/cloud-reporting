## 1. 프로젝트 개요

### 1.1 배경 및 목적

Cloud Ops Team은 매월 클라우드 비용 리포트를 수작업 엑셀로 생성·배포하고 있다. 이 과정에서 담당자의 반복 업무 부담, 데이터 정합성 오류, 배포 지연이 지속적으로 발생하여 경영진 및 실무자의 의사결정이 지연되고 있다. 본 프로젝트는 **엑셀 원천 데이터 기반 월별 리포트 자동 생성, 시각화, 다운로드 및 이메일 구독 발송을 하나의 웹 애플리케이션으로 통합**하여 운영 프로세스를 완전 자동화하는 것을 목적으로 한다.

### 1.2 비즈니스 목표

- 리포트 준비 시간 **80% 단축** (수작업 대비)
- 수기 입력 오류 **완전 제거** (오류 0건)
- 리포트 양식 **100% 표준화**
- 경영진·실무자의 비용 이상 탐지 및 의사결정 지원 수준 **High** 달성

### 1.3 현재 상황 (As-Is)

- 담당자가 부서별 엑셀 원천 데이터를 수작업으로 취합·정리
- 차트·피벗 등 시각화 요소를 매월 수동 생성
- 이메일 첨부 배포 시 수신자 명단 관리 및 발송 모두 수작업
- 컬럼명 변경 등 스키마 편차 발생 시 매핑 오류로 재작업 반복

### 4.2 기능 상세

#### 데이터 업로드 및 검증 (FR-01-01~05)

- **비즈니스 목적**: 부서별로 제공되는 형식이 다른 원천 엑셀 데이터를 오류 없이 시스템에 적재하기 위함

| 요구사항 ID | 기능명 | 우선순위 |
|------------|--------|----------|
| FR-01-01 | Excel 파일 업로드 (최대 12개월 Sheet 동시 처리) | 필수 |
| FR-01-02 | 대용량 파싱 엔진 (스트리밍 파싱, 비동기 처리) | 필수 |
| FR-01-03 | 컬럼 자동 매핑 (Alias 사전 기반 자동 인식) | 필수 |
| FR-01-04 | 데이터 정합성 체크 (유효성 검사, 오류/누락 경고) | 필수 |
| FR-01-05 | 스키마 검증 UI (컬럼 매핑 현황 시각 표시) | 권장 (Phase 2) |

- **사용자 흐름**:
  1. 담당자가 월별 엑셀 파일(최대 12개월 시트)을 업로드한다
  2. 시스템이 컬럼을 자동 인식하고 유효성을 검사한다
  3. 오류·누락 항목이 경고 메시지로 표시되고, 정상 시 다음 단계로 진행된다
- **입력 데이터**: 월별 클라우드 비용 원천 엑셀 파일 (최대 12개 시트)
- **출력 결과**: 컬럼 매핑 결과 및 오류/누락 경고 목록
- **비즈니스 규칙**:
  - 컬럼명이 변경되어도 Alias 사전으로 자동 인식
  - 향후 DS Payer 연동 시 수동 업로드 없이 데이터 자동 수집 가능해야 함

#### 템플릿 선택 — 리포트 라이브러리 (FR-02-01~03)

| 요구사항 ID | 기능명 | 우선순위 |
|------------|--------|----------|
| FR-02-01 | 리포트 라이브러리 (12종 카드형 목록, Top 6 확정 필요) | 필수 |
| FR-02-02 | 목적별 필터링 (유형/주기/부서 조건 필터링 및 검색) | 권장 |
| FR-02-03 | 템플릿 확장성 (Library 방식 신규 리포트 추가 구조) | 권장 |

- **비즈니스 목적**: 담당자가 원하는 리포트를 직관적으로 선택할 수 있도록 함
- **사용자 흐름**:
  1. 담당자가 카드형 목록에서 원하는 리포트 유형을 확인한다
  2. 유형·주기·부서 조건으로 필터링한다
  3. 원하는 카드를 선택하여 상세 팝업으로 진입한다
- **입력 데이터**: 필터 조건 (유형, 주기, 부서)
- **출력 결과**: 조건에 맞는 리포트 카드 목록
- **비즈니스 규칙**:
  - 총 12종 리포트 제공, 그 중 1차 MVP는 필수 Top 6 확정 필요
  - 신규 리포트를 지속 추가할 수 있는 Library 방식 구조 유지

> ⚠️ **미결 사항**: 필수 Top 6 리포트 최종 선정은 경영진·Cloud Ops 협의 필요 (의사결정 항목 #1)

#### 리포트 생성 (FR-03-01~06)

| 요구사항 ID | 기능명 | 우선순위 |
|------------|--------|----------|
| FR-03-01 | Cost & Usage 요약 (월별/일별 상세, Top 5 자동 추출, 요약 테이블 & 랭킹) | 필수 |
| FR-03-02 | Product/Region별 비용 (복합 차트: 막대+라인+**파레토**) | 필수 |
| FR-03-03 | Account & Forecast (Account별 누적비용, 예상 월 비용 Forecast) | 필수 |
| FR-03-04 | 집계 엔진 (다차원 Tag/Region 집계, Rank·추이·증감률 계산) | 필수 |
| FR-03-05 | Excel 템플릿 엔진 (서식/차트/피벗 자동 생성, Raw Data 병합, OpenXML) | 필수 |
| FR-03-06 | 웹 차트 렌더링 (ECharts 기반 동적 차트 미리보기) | 권장 |

- **비즈니스 목적**: 수작업 없이 표준화된 Excel 리포트(차트·피벗 포함)를 자동 생성
- **사용자 흐름**:
  1. 템플릿을 선택하면 시스템이 원천 데이터를 집계한다
  2. 설정된 차트·피벗 규칙에 따라 자동으로 Excel 파일이 생성된다
  3. Raw Data 시트가 함께 포함되어 2차 가공이 가능하다
- **입력 데이터**: 업로드된 원천 데이터, 선택된 템플릿
- **출력 결과**: Excel 리포트 파일 (서식·차트·피벗·Raw Data 포함)
- **비즈니스 규칙**:
  - 지원 리포트 유형: Cost & Usage 요약 / Product·Region별 비용 / Service별 사용 비용 / Tag별 현황 / Account & Forecast / 통합 내보내기
  - Product·Region별 / Service별 리포트: **복합 차트(막대+라인+파레토)** 시각화 포함
  - OpenXML 표준 준수, Windows·Mac·Mobile 뷰어 호환

#### 팝업 상세 및 즉시 다운로드 (FR-04-01~04)

| 요구사항 ID | 기능명 | 우선순위 |
|------------|--------|----------|
| FR-04-01 | 카드형 목록 → 모달 팝업 (설정 및 다운로드 완결 UI) | 필수 |
| FR-04-02 | 미리보기 (리포트 상세 정보 및 데이터 미리보기) | 필수 |
| FR-04-03 | 즉시 다운로드 (Excel 서식/피벗 포함, 월/형식 설정) | 필수 |
| FR-04-04 | 다운로드 이력 (Audit Log 기록 및 조회) | 권장 (Phase 2) |

- **비즈니스 목적**: 카드 선택부터 다운로드까지 팝업 내에서 완결, 이탈 없이 작업 완료
- **사용자 흐름**:
  1. 카드를 선택하면 모달 팝업이 열린다
  2. 리포트 상세 정보 및 데이터 미리보기를 확인한다
  3. 월·형식을 설정하고 즉시 다운로드한다
- **입력 데이터**: 선택 월, 내보내기 형식 설정
- **출력 결과**: Excel 파일 즉시 다운로드
- **비즈니스 규칙**: 다운로드 이력은 Audit Log에 기록 (Phase 2)

#### 구독 발송 (FR-05-01~04)

| 요구사항 ID | 기능명 | 우선순위 |
|------------|--------|----------|
| FR-05-01 | 자동 발송 스케줄러 (매월 10일 09:00, 휴일 순연 확정 필요) | 필수 |
| FR-05-02 | 구독자 리스트 관리 (등록/수정/삭제, 계정/부서별 권한) | 필수 |
| FR-05-03 | 주말/공휴일 예외처리 (발송일 자동 순연 로직) | 필수 |
| FR-05-04 | 재시도 및 실패 처리 (자동 재시도, 최종 실패 시 알림) | 필수 |

- **비즈니스 목적**: 경영진·실무자가 매월 리포트를 자동으로 수신하여 별도 요청 없이 현황 파악
- **사용자 흐름**:
  1. 관리자가 구독자 리스트(계정/부서별 권한 포함)를 등록·관리한다
  2. 매월 10일 09:00에 시스템이 자동으로 리포트를 이메일 발송한다
  3. 발송 실패 시 자동 재시도하고, 최종 실패 시 알림을 발송한다
- **입력 데이터**: 구독자 리스트, 발송 일정
- **출력 결과**: 이메일 자동 발송, 발송 성공/실패 알림
- **비즈니스 규칙**:
  - 발송일이 주말·공휴일인 경우 자동 순연
  - 순연 여부(익영업일)는 운영팀 최종 확정 필요

> ⚠️ **미결 사항**: 휴일 시 익영업일 순연 여부 최종 결정 필요 (의사결정 항목 #4)

## 7. MVP 정의

### 7.1 1차 출시 포함 기능

- **데이터 업로드 및 검증** — 자동화의 출발점, 없으면 전체 프로세스 불가
- **리포트 라이브러리 (Top 6 선정 후)** — 핵심 비즈니스 가치 제공
- **리포트 생성 (Excel 자동 생성)** — 80% 시간 단축의 핵심
- **팝업 상세 및 즉시 다운로드** — 리포트 소비 완결
- **구독 발송** — 정시 자동 배포로 배포 지연 문제 해결
- **목적별 필터링 및 검색** — 12종 리포트 탐색 UX 최소 요건

### 7.2 이후 버전 기능

- **스키마 검증 UI** — 운영 안정화 후 추가 (담당자 편의 개선)
- **다운로드 이력 Audit Log 조회** — 보안 요건 강화 시점에 추가
- **추가 리포트 7~12번** — 의사결정 완료 후 순차 추가
- **DS Payer 자동 연동** — DS Payer 환경 구축 이후 구현
