## 2. 배경 및 목적

### 2.1 추진 배경

- 수작업 엑셀 리포팅의 비효율성으로 인한 시간 낭비
- 데이터 정합성 오류 빈발로 리포트 신뢰성 저하
- 배포 지연 문제로 경영진 및 실무자의 의사결정 지연
- 반복적인 수기 작업에 따른 담당자 업무 부담 과중

### 2.2 프로젝트 목적

- 리포트 준비 시간 80% 단축
- 경영진 및 실무자 만족도 제고
- 운영 프로세스 완전 자동화
- 데이터 정합성 보장 및 표준화된 리포트 양식 제공

## 3. 서비스 범위 및 사용자 시나리오

### 3.1 End-to-End 자동화 프로세스 (6단계)

데이터 업로드부터 최종 구독 발송까지 사용자의 개입을 최소화하고 데이터 정합성을 보장하는 6단계 순환 구조로 운영됩니다.

| 단계 | 프로세스명 | 설명 |
|------|-----------|------|
| **01** | 데이터 업로드 | 월별 원천 데이터(Excel) 최대 12개월(Sheet) 대용량 파싱 엔진으로 업로드 |
| **02** | 매핑 및 검증 | 컬럼 자동 매핑, 데이터 정합성 체크, 오류/누락 경고 제공 |
| **03** | 템플릿 선택 | 리포트 라이브러리에서 카드형 UI로 목적별 템플릿 필터링 및 선택 |
| **04** | 리포트 생성 | Excel 포맷 변환, 피벗/차트 자동 구성, 원천 데이터 시트 병합 |
| **05** | 팝업 상세/다운로드 | 미리보기 및 정보 확인, 즉시 다운로드, 내보내기 설정(월/형식) |
| **06** | 구독 발송 | 매월 10일 자동 발송, 구독자 리스트 관리, 주말/공휴일 예외 처리 |

## 6. 시스템 아키텍처 요구사항

기반 플랫폼: **Spring Boot + React SPA + PostgreSQL + Docker**

### 6.1 프론트엔드 레이어 (React SPA)

| 컴포넌트 | 기능 | 상세 |
|---------|------|------|

| 컴포넌트 | 기능 | 상세 |
|---------|------|------|
